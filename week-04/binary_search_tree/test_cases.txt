Binary Search Tree Test Cases
==============================

Test Case 1: Insert in order
Input: Insert [15, 12, 13, 11, -1, 0, 101]
Inorder Output: -1 0 11 12 13 15 101

Test Case 2: Insert reverse order
Input: Insert [7, 6, 5, 4, 3, 2, 1]
Inorder Output: 1 2 3 4 5 6 7

Test Case 3: Insert with duplicates (should not add)
Input: Insert [5, 3, 7, 3, 5]
Inorder Output: 3 5 7

Test Case 4: Single node
Input: Insert [42]
Inorder Output: 42

Test Case 5: Balanced tree
Input: Insert [50, 30, 70, 20, 40, 60, 80]
Inorder Output: 20 30 40 50 60 70 80

Test Case 6: Left skewed tree
Input: Insert [5, 4, 3, 2, 1]
Inorder Output: 1 2 3 4 5

Test Case 7: Right skewed tree
Input: Insert [1, 2, 3, 4, 5]
Inorder Output: 1 2 3 4 5

Test Case 8: Random insertions
Input: Insert [100, 50, 150, 25, 75, 125, 175]
Inorder Output: 25 50 75 100 125 150 175
